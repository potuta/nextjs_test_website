## 🧩 Overview

TaskManager is a simple multi-user to-do list application built to understand full-stack development with Next.js. Each authenticated user has their own private task list that only they can access and manage.

This project was mainly built as a hands-on exercise to explore authentication, database integration, and deployment workflows.

### ⚙️ Key Features
* 🔐 User authentication and session handling
* 🧑‍💼 Per-user isolated task lists
* ➕ Create, update, and delete tasks
* 📋 Personal dashboard for each user
* ☁️ Deployed on Vercel
* 🗄️ PostgreSQL database integration (Neon)

### 🛠️ Tech Stack Used
* Next.js (App Router)
* TypeScript
* Prisma ORM
* PostgreSQL (Neon)
* Better Auth
* Tailwind CSS + shadcn/ui
  
### 🧠 What This Project Helped Me Learn

This project helped reinforce key full-stack concepts, including:

* Authentication flow in modern web apps
* Database modeling for multi-user systems
* Server-side vs client-side rendering in Next.js
* Deployment differences between local and production environments
* Working with environment variables and external services

## 📚 Next.js (App Router)

I explored the modern Next.js ecosystem and built a full-stack application using the App Router. Coming from a .NET Core MVC background, I focused on understanding how similar architectural concepts translate into the React/Next.js environment.

### ⚙️ Tech Stack

* **TypeScript** – Strongly typed development across frontend and backend
* **Tailwind CSS** – Utility-first styling for rapid UI development
* **PostgreSQL** – Relational database for persistent storage

### 🗄️ Database & ORM

* **Prisma ORM**

  * Schema modeling and migrations
  * Querying relational data
  * Integrating with Next.js server components and route handlers

### 🎨 UI & Components

* **shadcn/ui**

  * Leveraged pre-built, accessible UI components
  * Customized layouts including sidebar templates
* **lucide-react**

  * Integrated consistent and scalable iconography

### 🔐 Authentication & Authorization

* **better-auth**

  * Implemented sign-up and sign-in flows
  * Managed sessions via database and browser storage
* **Custom RBAC & PBAC**

  * Designed and implemented dynamic role-based and permission-based access control
  * Overcame limitations of static role definitions

### 🧾 Forms & Validation

* **React Hook Form**

  * Efficient form state management
* **Zod**

  * Schema-based validation for type-safe input handling

### 🚦 Next.js Core Concepts

* App Router and file-based routing
* Server and Client Components
* Route Handlers for backend logic
* Data fetching strategies and rendering patterns

### 💻 Developer Tooling

* CLI workflows (`npm run dev`, Prisma commands, etc.)
* Environment configuration and project setup

---

### 🧠 Key Takeaway

Transitioning from .NET Core MVC to Next.js highlighted strong parallels in full-stack architecture, while introducing modern patterns such as server components, hybrid rendering, and a more integrated frontend-backend development experience.
